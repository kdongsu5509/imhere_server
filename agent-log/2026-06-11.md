### Failures & Root Causes
| 시도 | 실패 원인 |
|------|---------|
| `write_to_file` 호출 시 프로젝트 정적 리소스 파일인 `admin.css` 생성 과정에서 `ArtifactMetadata`를 명시함 | `ArtifactMetadata`는 오직 아티팩트 전용 디렉토리 하위의 파일에만 쓸 수 있으므로 validation 에러 발생 |
| 템플릿 리팩토링 후 Gradle 테스트가 `SecurityConfig` 빈 초기화 에러로 빌드 실패 | 기존 로컬 워크스페이스 내 Kotlin 파일들에 잘못 수정된 상태(순환 의존 및 `@param:Value` 오용)가 남아있어, 템플릿 컴파일 시점에 Kotlin recompile이 촉발되며 기존 잠재 버그가 드러남 |
| Kotlin 리소스를 HEAD 상태로 restore하였으나 `admin.allowed-ips` 프로퍼티 누락으로 테스트 구동 실패 | HEAD의 `SecurityConfig`에 선언된 `allowed-ips`에는 기본값이 없었으나, 테스트 설정 파일(`application-test.yml`)에는 이 속성이 정의되어 있지 않아 placeholder resolution 오류 발생 |

- `write_to_file` 호출 시 프로젝트 소스 코드 파일에서는 `ArtifactMetadata` 지정을 생략하여 생성
