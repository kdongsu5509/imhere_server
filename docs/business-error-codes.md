# 📑 비즈니스 에러 코드 명세 (Detailed Business Error Codes)

본 문서는 프로젝트의 세부 비즈니스 에러 코드를 정리한 명세입니다. 모든 에러는 `BaseErrorCode` 인터페이스를 구현한 Enum으로 관리됩니다.

## 📌 에러 코드 체계 (Naming Convention)
비즈니스 코드는 `[DOMAIN]-[INDEX]` 형식을 따르며, `INDEX`의 앞자리는 대응하는 HTTP 상태 코드의 카테고리를 나타냅니다.

*   **0xx**: 400 Bad Request
*   **1xx**: 401 Unauthorized
*   **2xx**: 403 Forbidden
*   **3xx**: 404 Not Found
*   **4xx**: 405 Method Not Allowed
*   **5xx**: 409 Conflict
*   **6xx**: 415 Unsupported Media Type
*   **7xx**: 422 Unprocessable Entity
*   **9xx**: 500 Internal Server Error

## 🌐 공통 에러 (CommonErrorCode)
| 비즈니스 코드 | HTTP 상태 | 메시지 |
| :--- | :--- | :--- |
| `GLOBAL-000` | 400 | 잘못된 요청입니다. 입력값을 확인해주세요. |
| `GLOBAL-001` | 400 | 요청 바디를 읽을 수 없거나 형식이 잘못되었습니다. |
| `GLOBAL-100` | 401 | 인증에 실패했습니다. 다시 로그인해주세요. |
| `GLOBAL-200` | 403 | 해당 리소스에 접근할 권한이 없습니다. |
| `GLOBAL-300` | 404 | 요청하신 리소스를 찾을 수 없습니다. |
| `GLOBAL-400` | 405 | 지원하지 않는 HTTP 메서드입니다. |
| `GLOBAL-500` | 409 | 이미 존재하는 데이터이거나 상태가 충돌합니다. |
| `GLOBAL-600` | 415 | 지원하지 않는 미디어 타입입니다. |
| `GLOBAL-700` | 422 | 요청을 처리할 수 없습니다. |
| `GLOBAL-900` | 500 | 외부 서비스와의 통신 중 오류가 발생했습니다. |
| `GLOBAL-901` | 500 | 서버 내부 오류가 발생했습니다. |
